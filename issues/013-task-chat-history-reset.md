# 013: Task - チャット履歴クリア（リセット）機能の実装

## 概要

ユーザーがWebUIおよびCLIのREPLから、現在のチャット履歴を簡単にクリアし、新しい会話を開始できるようにする機能を実装します。

## 関連仕様書

- [チャット履歴クリア（リセット）機能 仕様書](../doc/chat_history_reset_spec.md)

## 受け入れ基準 (Acceptance Criteria)

- **UI (Gradio)**
  - [ ] WebUIに「新しいチャット」または「リセット」ボタンが配置されている。
  - [ ] ボタンをクリックすると、チャット履歴表示がクリアされ、内部の履歴データもリセットされる。
  - [ ] 履歴クリア後も、LLMプロバイダー、モデル、システムプロンプトなどの設定は維持されている。
- **CLI (REPL)**
  - [ ] REPLで `/reset` コマンドが利用できる。
  - [ ] `/reset` コマンドを実行すると、内部の履歴データがリセットされる。
  - [ ] 履歴クリア後も、LLMプロバイダー、モデル、システムプロンプトなどの設定は維持されている。
- **共通**
  - [ ] 新機能に関するユニットテストが追加され、すべてのテストがパスする。

## タスク分割 (Sub-tasks)

1.  **Chat Logic**: `chat_logic.py` に、チャット履歴をクリアするためのメソッドを追加する。
2.  **Web UI**: `webui.py` に「新しいチャット」ボタンを追加し、クリック時に `chat_logic` の履歴クリアメソッドを呼び出すイベントハンドラを実装する。
3.  **CLI**: `cli.py` のREPLに `/reset` コマンドのパーサーと、`chat_logic` の履歴クリアメソッドを呼び出すロジックを実装する。
4.  **Testing**: `tests/` ディレクトリ内の関連テストファイル（`test_chat_logic.py`, `test_webui.py`, `test_cli.py` 等）に、履歴クリア機能の動作を検証するユニットテストを追加する。
5.  **Documentation**: `README.md` や関連ドキュメントを更新し、新しい履歴クリア機能の使い方を説明する。
