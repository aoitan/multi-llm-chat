# Task: Update system prompts to enforce TDD workflow

## 概要
開発フローを TDD へ移行するため、`AGENTS.md` および `.gemini/GEMINI.md` に「テストを先に書き、赤→緑→リファクタの手順を踏む」指針を明文化し、各 LLM エージェントが一貫したワークフローに従うよう更新する。

## 作業詳細
- `AGENTS.md` の「Testing Guidelines」節を中心に、以下を盛り込む:
  - 実装前に failing test を書くことを必須化する。
  - 1 つの機能は小さなサイクルで「Red → Green → Refactor」を回す。
  - PR 説明やコミット時に、追加したテストケースを列挙することを推奨 / 必須化。
- `.gemini/GEMINI.md` に運用ルールを追加し、LLM への指示として:
  - 「作業開始時にテスト計画を述べ、テストが先に書かれているか確認する」等、TDD 運用に従う旨を日本語で明記。
  - 実装ステップごとに失敗するテスト → 実装 → テストパス報告を求める。
- 必要に応じて `README.md` や関連ドキュメントにも TDD への方針転換を追記し、コントリビュータが迷わないようにする。

## 完了条件
- `AGENTS.md` と `.gemini/GEMINI.md` に、TDD を必須とする具体的な手順と開発サイクルが明記されている。
- ドキュメントを読めば「テストを先に書かないといけない」「PR ではテストを説明する」という期待値が分かる。
- レビュワーが TDD フロー遵守を確認できる運用（チェックリスト等）が文書化されている。
