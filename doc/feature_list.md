# 機能リスト一覧

Multi-LLM Chatが提供する機能をカテゴリ別に一覧化しています。

## 1. コア機能
*   **マルチLLMチャット**: Gemini と ChatGPT の2つのモデルと同時に、または個別に会話が可能。
*   **統一コンテキスト**: 複数のLLMとの会話履歴を共有し、文脈を維持した対話が可能。
*   **メンションルーティング**: `@gemini`, `@chatgpt`, `@all` を使って応答するAIを指定可能。
*   **思考メモ機能**: メンションなしの入力はAPI呼び出しを行わず、履歴にのみ記録（思考整理用）。
*   **APIキー管理**: 環境変数 (`.env`) からのセキュアな読み込み。

## 2. インターフェース
### Web UI (Gradio)
*   **チャット画面**: モダンで直感的なチャットインターフェース。
*   **ストリーミング応答**: AIの回答をリアルタイムで表示。
*   **履歴管理パネル**:
    *   **User ID設定**: ユーザーごとの履歴分離。
    *   **保存**: 現在の会話に名前を付けて保存。
    *   **読み込み**: 保存済み履歴のリスト表示と読み込み。
    *   **新規作成**: 会話のリセットと新規開始。
*   **システムプロンプト設定**: 画面上部からAIの役割や挙動を動的に変更可能。
*   **トークン数表示**: 現在のコンテキスト長をリアルタイム表示。

### CLI (Command Line Interface)
*   **対話シェル**: コマンドラインでの軽量なチャット。
*   **システムコマンド**:
    *   `/system`: システムプロンプトの確認・設定・クリア。
    *   `/history`: 履歴操作（save, load, list, new, export）。
*   **MCPツール実行**: 起動時に設定したMCPサーバーのツールを利用可能。

## 3. 拡張機能
*   **自律エージェント (ReAct)**: LLMが自律的に思考し、ツールを実行して回答を生成するループ機能。
*   **MCP (Model Context Protocol) 対応**:
    *   外部ツール（天気予報、Web検索、DB操作など）との標準化された接続。
    *   自動ツール選択と実行結果の統合。
*   **コンテキスト圧縮**:
    *   **自動プルーニング**: 最大トークン数を超えないよう、古い履歴を自動的に削除（スライディングウィンドウ）。
    *   **システムプロンプト保護**: 圧縮時もシステム設定は常に維持。

## 4. 開発・運用機能
*   **3層アーキテクチャ**: UI、ビジネスロジック、コアロジックの分離による高い保守性。
*   **TDDワークフロー**: テスト駆動開発に基づいた堅牢な設計。
*   **Dockerサポート**: コンテナによる実行環境の統一（※要確認・計画中）。
