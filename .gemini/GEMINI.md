## TDD ワークフロー（必須）

このプロジェクトでは、[開発ワークフロー (`doc/development_workflow.md`)](doc/development_workflow.md) に基づいたテスト駆動開発（TDD）を必須としています。以下の手順に従ってください：

### 作業開始前
1. `issues` ディレクトリから担当する **Task** を選択し、実装する機能の**テスト計画を述べる**（どのようなテストケースを書くか）
2. **失敗するテストを先に書く**（Red フェーズ）
3. `uv run pytest` を実行し、テストが正しく失敗することを確認

### 実装フェーズ
1. テストを通すための**最小限の実装を追加**（Green フェーズ）
2. `uv run pytest` を実行し、すべてのテストが通ることを確認
3. 必要に応じてリファクタリング（Refactor フェーズ）

### コミット・PR 作成時
- コミットは **Task** 単位で行い、メッセージに**追加したテストケース名を列挙**する
  例: `feat(core): テンプレート機能実装 - test_template_rendering を追加`
- PR 説明に「追加したテスト」セクションを必ず含める
- TDD サイクルをスキップした場合は、その理由を明記する

### 禁止事項
- テストを書く前に実装を進めること
- テストが失敗していることを確認せずに実装すること
- テストケースを列挙せずにPRを作成すること

**例外**: 緊急のホットフィックスやドキュメント専用の変更では TDD スキップ可能ですが、PR に明示的な理由を記載してください。
